get_product_links(sort: Optional[int] = None, c_category: Optional[int] = None, category: Optional[int] = None, gender: Optional[str] = None, designer: Optional[int] = None, pages: int = 0, proxies: Optional[dict] = None, headers: Optional[dict] = None, save: bool = False)

this function creates all the product links that we intend to scrape and returns them in a list.

Parameters

  • sort (int, optional) – Atribute of the link. It sorts the links and can have the following values: 1 - high price to low price 2 - newest first 3 - their own picks 4 - low price to high price None - it defaults to 3

  • c_category (int, optional) – Atribute of the link. It filters the links to only get items within the specified categories. There is many possible values to this parameter and to see how to get them refer to the demo package in the github repo. if None, it wont filter the results by category.

  • category (int, optional) – Atribute of the link. It filters the links to only get items within the specified sub-categories. There is many possible values to this parameter and to see how to get them refer to the demo package in the github repo. if None, it wont filter the results by sub-category.

  • gender (str, optional) – Atribute of the link. It filters the links to only get items within the specified gender. It can have the following values: ‘men’ - only men products ‘women’ - only women products ‘kids’ - only kids products None - it wont filter the results by gender

  • category – Atribute of the link. It filters the links to only get items within the specified brand. There is many possible values to this parameter and to see how to get them refer to the demo package in the github repo. if None, it wont filter the results by sub-category.

  • pages (int, optional) – Atribute of the link. The number of pages thats desired to scrape.

  • proxies (dict, optional) – This parameter is usefull in case you want to use an external proxie to send the request. Usually is used when atempting to do proxy rotation. defaults to None.

  • headers (dict, optional) – This parameter is usefull in case you want to use a different header to send the request. Usually is used when atempting to do header rotation. defaults to None., defaults to None

  • save (bool, optional) – if True it saves all the scraped links to a csv file (named product_links.csv) in the data folder. Defaults to False.

Returns

Returns a list with all the product links

Return type

list
